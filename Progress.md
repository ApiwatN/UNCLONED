# 🚀 UNCLONED Finalization Roadmap & Progress

จากการวิเคราะห์ระบบและการอัปเดตครั้งใหญ่ในวันนี้ นี่คือความคืบหน้าของแผนงาน 5 ข้อเพื่อดันโปรเจกต์ UNCLONED ให้ขึ้นหิ้งเป็นเว็บ E-Commerce มาตรฐานสากลครับ:

## ✅ 1. การนำเว็บไซต์ขึ้นระบบจริง (Deployment & Vercel)

- **สถานะ:** เสร็จสมบูรณ์ (Completed)
- **กระบวนการ:** ทำการอัปโหลดโค้ดขึ้น GitHub และเชื่อมต่อเข้ากับ Vercel สำเร็จ จัดการเปลี่ยน Framework Preset เป็น Next.js และใส่ Environment Variables ทุกตัว (Supabase + Cloudinary) ทำให้เว็บไซต์สามารถทดลองรันจริงบนโลกออนไลน์ได้แล้ว

## ✅ 2. 🔍 การทำ SEO เชิงลุก (Global SEO Optimization)

- **สถานะ:** เสร็จสมบูรณ์ (Completed)
- **กระบวนการ:**
  - อัปเกรด `generateMetadata` ในโฟลเดอร์ `/product/[id]/page.tsx`
  - เมื่อแชร์ลิงก์สินค้าไปที่ Facebook/LINE จะปรากฏรูปและรายละเอียดของสินค้านั้นเป๊ะๆ ไม่ใช่หน้าแหว่ง
  - ติดตั้ง `JSON-LD Schema (Product)` ไว้คุยกับหุ่นยนต์ Google เพื่อรองรับการแสดงผล Rich Snippets (โชว์ราคา และ โชว์ป้ายว่า In-stock ในหน้า Google Search ได้เลย)

## ✅ 3. 📈 การติดตั้งระบบติดตามลูกค้า (Analytics & Conversion Tracking)

- **สถานะ:** เสร็จสมบูรณ์ (Completed)
- **กระบวนการ:**
  - ฝัง Component `TrackingScripts.tsx` เตรียมพร้อมรับไอดีใช้งาน Google Analytics 4 (GA4) และ Meta (Facebook) Pixel ซึ่งใช้ได้ฟรี!
  - ติดตั้ง Event Action ฝังเข้าไปตอนคลิกปุ่ม **"หยิบใส่ตะกร้า"** (Add to Cart events) เพื่อสูบข้อมูลสินค้าและมูลค่าที่กำลังจะซื้อ ส่งกลับไปกักตุนเป็น Data เตรียมไว้ยิงแอด Retargeting

## ✅ 4. 🛡️ การรักษาความปลอดภัยฐานข้อมูล (Security & RLS)

- **สถานะ:** เสร็จสมบูรณ์ทางฝั่งโค้ด หน้าที่แอดมินนำ SQL ไปรัน
- **กระบวนการ:**
  - ปรับแก้ API Admin ทั้งหมดให้ใช้สิทธิสูงสุด (Service Role Key)
  - ออกสคริปต์ SQL (`setup_rls_security.sql`) สำหรับล็อกตารางสินค้าและออเดอร์ด้วยมาตรการ Row Level Security (RLS) เพื่อกันคนนอกแอบยิง API เข้ามาแก้ไขข้อมูลสต็อกสินค้า หรือแอบดูออเดอร์ของคนอื่น

## ✅ 5. 🛍️ การเร่งการตัดสินใจซื้อของลูกค้า (Retail UX & Animations)

- **สถานะ:** เสร็จสมบูรณ์ (Completed)
- **กระบวนการ:**
  - ดึงข้อมูลสต็อกแบบเรียลไทม์มาเช็ค หากระบุว่าสินค้าไซส์ใดเหลือ < 5 ชิ้น ระบบจะขึ้นป้าย **"🔥 รีบหน่อย! ไซส์ ... เหลือเพิ่ง ... ชิ้นเท่านั้น"** ขึ้นมาโชว์ที่หน้าการ์ดสินค้า และกระพริบใต้ชื่อสินค้าในหน้ารายละเอียด เพื่อสร้างความรู้สึกกลัวพลาด (FOMO) เร่งรัดการซื้อให้จบไวขึ้น

## 🔮 แผนงานในอนาคต (Future Roadmap)

### 📌 การดึงสถิติยอดเข้าชมเว็บมาแสดงในหน้าแรกของระบบ Admin (Google Analytics Data API)
* **โจทย์:** ปัจจุบันแอดมินต้องเปิดดูข้อมูลเชิงลึกเรื่องคนเข้าเว็บ (Traffic, Demographics, Campaign Success) ผ่านแพลตฟอร์มของ Google Analytics (GA4) โดยตรง
* **แนวทางการพัฒนา:** 
  - เขียน API Backend (ใช้บริการ Oauth & Google Cloud Service Account) ไปดึงข้อมูลจากระบบ Google Analytics Data API
  - ดึงข้อมูลขั้นพื้นฐาน เช่น จำนวนคนเข้าชมในวันนี้, ช่องทางที่คนเข้ามามากที่สุด (Traffic Source), ระยะเวลาเฉลี่ยที่คนอยู่บนเว็บ
  - นำข้อมูลเหล่านี้มาปั้นเป็น Chart ให้แสดงผลอยู่ในหน้า `/admin/analytics` ของเราเอง เพื่อรวบยอดข้อมูลให้ผู้บริหารหรือเจ้าของกิจการสามารถดู "ยอดคลิก" คู่กับ "ยอดขายจริง" ได้ในจอเดียวโดยไม่ต้องสลับแพลตฟอร์ม
